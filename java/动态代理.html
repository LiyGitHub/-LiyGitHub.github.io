# Java动态代理

Java动态代理是一种实现 AOP（面向切面编程）的方法之一。代理类不是在Java代码中手动编写，而是在运行时通过Java反射动态生成的。

# 什么是代理

代理是一种设计模式，它提供了对其他对象的访问控制。代理有两种形式：静态代理和动态代理。静态代理需要手动编写代理类，而动态代理则是在运行时通过反射动态生成的。

# 什么是动态代理

动态代理是在运行时通过反射动态生成的代理类。它可以在运行时动态地创建代理对象，而不需要手动编写代理类。

动态代理有两种实现方式：JDK动态代理和CGLIB动态代理。其中，JDK动态代理只能代理实现了接口的类，而CGLIB动态代理则可以代理任意的类。

# JDK动态代理

JDK动态代理是Java提供的一种动态代理方式。它可以在运行时通过反射动态地创建代理对象，而不需要手动编写代理类。

以下是使用JDK动态代理创建代理对象的示例代码：

public class JdkDynamicProxyExample {

    public static void main(String[] args) {
        Subject subject = new RealSubject();
        InvocationHandler handler = new MyInvocationHandler(subject);
        Subject proxy = (Subject) Proxy.newProxyInstance(subject.getClass().getClassLoader(),
                subject.getClass().getInterfaces(), handler);
        proxy.request();
    }

    interface Subject {
        void request();
    }

    static class RealSubject implements Subject {
        @Override
        public void request() {
            System.out.println("RealSubject: request()");
        }
    }

    static class MyInvocationHandler implements InvocationHandler {

        private final Object target;

        public MyInvocationHandler(Object target) {
            this.target = target;
        }

        @Override
        public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
            System.out.println("before " + method.getName() + "()");
            Object result = method.invoke(target, args);
            System.out.println("after " + method.getName() + "()");
            return result;
        }
    }
}
在以上代码中，我们创建了一个 Subject 接口和一个实现了 Subject 接口的类 RealSubject。然后，我们创建了一个实现了 InvocationHandler 接口的类 MyInvocationHandler，在这个类中我们可以通过反射来调用 RealSubject 中的方法。最后，我们使用 Proxy.newProxyInstance() 方法来创建代理对象，并调用 proxy.request() 方法来使用代理对象。

CGLIB动态代理
CGLIB动态代理是另一种动态代理方式。它通过创建被代理类的子类来实现代理。由于它是通过继承来实现代理的，因此可以代理任意的类。
